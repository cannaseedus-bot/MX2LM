# MX2LM Manifesto

## What MX2LM Is

* A canonical cognitive runtime specification.
* Frozen runtime law paired with formal schemas, conformance vectors, and proofs.
* A language for atomic composition across blocks, bindings, CSS, and verifier outputs.

## What MX2LM Is Not

* Not a tutorial, starter kit, or scaffold.
* Not an implementation guide, cookbook, or operational checklist.
* Not a staging ground for product glue, ad-hoc scripts, or mutable norms.

## Repository Posture

* Authoritative content is explicitly marked as law or specification.
* Reference artifacts (including the atomic runtime appendix) are non-authoritative and exist only to demonstrate executability.
* No instructions for running, wiring, or deploying are carried in-repo.

## Commitment

MX2LM remains a declarative, disciplined release: proofs stay tight, schemas stay normative, and examples stay quarantined as reference material only.
