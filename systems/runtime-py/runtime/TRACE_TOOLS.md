# MX2LM Trace Tools

This folder now ships small, deterministic utilities for structured traces generated by the π evaluator.

## 1) Persisting traces with shell artifacts

`runtime/trace_store.mjs` verifies the π trace seal against the rendered shell SVG and writes both to `runtime/traces/`.

Example:
```bash
node -e "import {persistTraceRecord} from './runtime/trace_store.mjs'; \
import fs from 'node:fs/promises'; \
const trace = JSON.parse(await fs.readFile('trace.json','utf8')); \
await persistTraceRecord({ trace, shellPath: 'rendered_shell.svg' });"
```

Guarantees:
- Trace seals are recomputed with shell hash before writing.
- Existing records must match or the write is rejected (immutable archive).
- Shell artifacts are copied next to the trace JSON for review.

## 2) Inspecting a stored trace

`runtime/trace_inspector.mjs` loads a stored record, re-verifies seals, and emits an HTML review file that includes the SVG geometry.

Example:
```bash
node runtime/trace_inspector.mjs runtime/traces/TRACE_ID.trace.json --output runtime/traces/TRACE_ID.html
```

Outputs:
- Validation status (seal + hash agreement).
- Human-readable trace payload (inputs, effects, outputs).
- Inline SVG geometry for visual inspection.
